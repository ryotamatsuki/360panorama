## ホットスポット設計

各地点はPannellumのtour機能を使い、`links` から `type: "scene"` の `hotSpots` を生成します。一本道のため、原則として前方リンクと後方リンクを1つずつ持ちます。

- N001: 次地点のみ
- N002-N023: 前地点と次地点
- N024: 前地点のみ

`src/tour-data.js` のリンクには以下を設定します。

- `direction`: `"forward"` または `"back"`
- `yaw`: 現在地点でホットスポットを表示する左右方向
- `pitch`: 現在地点でホットスポットを表示する上下方向
- `targetYaw`: 移動先地点で最初に向く左右方向
- `targetPitch`: 移動先地点で最初に向く上下方向
- `icon`: 表示方向を示す補助情報

初期値は、前方リンクを `yaw: 0` / `targetYaw: 0`、後方リンクを `yaw: 180` / `targetYaw: 180`、地面付近のマーカーとして `pitch: -8` にしています。

## ホットスポット調整手順

1. ローカルサーバーで起動する
2. 各地点を開く
3. 次地点へ進むべき方向を確認する
4. `yaw` を調整する
5. ホットスポットの上下位置が不自然なら `pitch` を調整する
6. 移動後の向きが不自然なら `targetYaw` を調整する
7. 戻り方向も同様に確認する

本番パノラマ画像の向きが地点ごとに統一されていない場合は、`targetYaw: "sameAzimuth"` も候補です。ただし現在は、画像差し替え後の手動調整を前提に固定値を入れています。

## 画像差し替えメモ

現在は24地点のルートを先に見せるため、既存の8枚のプロトタイプパノラマ画像を複数地点で共有しています。本番運用では、24地点分の2:1 equirectangular panoramaとサムネイルに差し替えてください。

Pannellum用画像の推奨条件:

- 形式: WebP、PNG、JPEG
- アスペクト比: 2:1
- 推奨サイズ: 4096 x 2048 または 2048 x 1024
- 投影形式: equirectangular panorama
